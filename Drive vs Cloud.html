We Need Cloud for Scalable Medicaid Analytics & Automation

Our pipeline must handle data growth, cross-table joins, and multi-user access. Even with ~100 GB today (local 64 GB RAM), cloud is the correct foundation for reliable performance, repeatable automation, and future expansion.

Purpose (What Cloud Enables)

Click to hide

Build a robust, automated validation + analytics system that remains fast and reliable as we scale beyond single-machine limits and shared-drive bottlenecks.

  • Automation: scheduled SAS jobs for monthly loads + QA test cases (repeatable runs).
  • Performance: faster I/O and predictable runtime for large joins and scans.
  • Scale: resize compute (RAM/CPU) instantly as data grows.
  • Collaboration: multiple users can run workflows without “my PC is busy” issues.
  • Reproducibility: same environment + versions → consistent outputs.
  • Audit trails: log runs, inputs, outputs, QC results for stakeholders.
  • Disaster recovery: snapshots/backups reduce risk of loss/corruption.
  • Standard delivery: publish curated outputs for dashboards/reporting.

Advantages (Cloud)

  • Scales on demand: increase RAM/CPU only when needed (pay-as-you-go).

  • Parallel pipelines: dev/test/prod runs without blocking others.
